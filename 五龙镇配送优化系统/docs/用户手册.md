## 系统概述

### 功能特点

五龙镇配送优化系统是一个专业的物流配送路径优化工具，主要特点包括：

- ✅ **智能优化**: 采用遗传算法进行路径优化
- ✅ **多模式配送**: 支持车辆和无人机协同配送
- ✅ **时间约束**: 考虑配送时效性要求
- ✅ **成本控制**: 精确计算各类成本
- ✅ **可视化展示**: 直观的图表和网络展示
- ✅ **数据驱动**: 支持多种数据格式输入

### 应用场景

- 农村电商物流配送优化
- 应急物资配送规划
- 快递末端配送方案设计
- 物流成本效益分析

### 系统要求

- **操作系统**: Windows 10+ / macOS 10.15+ / Ubuntu 18.04+
- **Python版本**: 3.8 或更高
- **内存**: 至少4GB RAM
- **存储空间**: 至少1GB可用空间

## 安装指南

### 步骤1: 下载系统

从项目仓库下载完整的系统包：

```bash
git clone <repository-url>
cd 五龙镇配送优化系统
```

### 步骤2: 安装Python环境

确保系统已安装Python 3.8或更高版本：

```bash
python --version
```

如果未安装，请从 [python.org](https://python.org) 下载安装。

### 步骤3: 安装依赖包

```bash
pip install -r requirements.txt
```

### 步骤4: 验证安装

运行测试命令验证安装是否成功：

```bash
python main.py --help
```

## 快速开始

### 基本运行

最简单的使用方式：

```bash
python main.py
```

系统将使用默认配置和数据文件运行优化，并将结果保存到 `output/` 目录。

### 指定输出目录

```bash
python main.py -o my_results/
```

### 使用自定义配置

```bash
python main.py -c config/my_config.json -o my_results/
```

## 配置管理

### 配置文件格式

配置文件采用JSON格式，主要包含以下部分：

```json
{
  "system_info": {
    "name": "五龙镇配送优化系统",
    "version": "1.0.0"
  },
  "vehicle_params": {
    "num_vehicles": 3,
    "capacity": 1500,
    "speed": 50
  },
  "drone_params": {
    "max_payload": 80,
    "speed": 60
  },
  "ga_params": {
    "population_size": 150,
    "generations": 500
  }
}
```

### 主要配置参数

#### 车辆配置 (`vehicle_params`)

| 参数               | 说明     | 默认值 | 单位 |
| ------------------ | -------- | ------ | ---- |
| `num_vehicles`     | 车辆数量 | 3      | 台   |
| `capacity`         | 载重能力 | 1500   | kg   |
| `speed`            | 行驶速度 | 50     | km/h |
| `fuel_consumption` | 油耗     | 0.252  | L/km |
| `fuel_price`       | 油价     | 6.89   | 元/L |

#### 无人机配置 (`drone_params`)

| 参数               | 说明     | 默认值 | 单位 |
| ------------------ | -------- | ------ | ---- |
| `max_payload`      | 最大载重 | 80     | kg   |
| `max_range`        | 最大航程 | 12     | km   |
| `speed`            | 飞行速度 | 60     | km/h |
| `battery_capacity` | 电池容量 | 2.132  | kWh  |

#### 算法配置 (`ga_params`)

| 参数              | 说明     | 默认值 | 范围     |
| ----------------- | -------- | ------ | -------- |
| `population_size` | 种群大小 | 150    | 50-500   |
| `generations`     | 迭代代数 | 500    | 100-2000 |
| `mutation_rate`   | 变异率   | 0.25   | 0.1-0.5  |
| `crossover_rate`  | 交叉率   | 0.85   | 0.5-0.95 |

## 数据准备

### 必需数据文件

1. **物流车配送距离.xlsx**

   - 包含各地点间的车辆行驶距离
   - 格式：第一列为起点，其他列为目的地
2. **五龙镇各村距离矩阵.xlsx**

   - 村庄间的距离数据
   - 用于计算无人机航程

### 数据格式要求

#### 距离矩阵格式

```
     A     B     C     D
A    0    10.5  15.2  8.7
B   10.5   0    12.1  9.3
C   15.2  12.1   0   18.6
D    8.7   9.3  18.6   0
```

#### 需求数据格式

系统内置了五龙镇各村庄的需求数据，也可以通过配置文件自定义：

```json
{
  "demands_piece": {
    "村庄A": 100,
    "村庄B": 150,
    "村庄C": 80
  }
}
```

## 结果解读

### 输出文件说明

运行完成后，系统会在输出目录生成以下文件：

1. **可视化图表**

   - `配送网络图.png`: 配送路径和网络结构
   - `成本构成图.png`: 各类成本占比
   - `配送时间分析.png`: 时间分布和时效分析
   - `优化历史.png`: 算法收敛过程
   - `需求分布图.png`: 需求量分布情况
2. **数据文件**

   - `optimization_results.json`: 详细的优化结果
   - `system.log`: 运行日志记录

### 结果分析

#### 配送网络图解读

- **红色线条**: 配送中心到五龙镇的主干线路
- **彩色线条**: 各车辆的配送路径
- **紫色线条**: 无人机配送航线
- **圆点**: 配送点位置
- **箭头**: 行驶方向

#### 成本构成分析

总成本主要包括：

- **车辆燃油费**: 按行驶距离和油耗计算
- **车辆时间成本**: 司机工时费用
- **车辆固定成本**: 车辆出车固定费用
- **无人机电费**: 飞行能耗费用
- **无人机固定成本**: 无人机使用固定费用

#### 时间约束评估

- **时间阈值**: 默认2小时
- **准时率**: 在阈值内完成的配送比例
- **模糊评估值**: 综合考虑平均时间、最大时间和准时率的评估指标

### 性能指标

系统会输出以下关键指标：

1. **优化目标值**: 总配送成本（元）
2. **收敛代数**: 算法达到最优解的迭代次数
3. **计算时间**: 优化过程耗时（秒）
4. **车辆利用率**: 实际使用的车辆比例
5. **无人机使用率**: 使用无人机的配送点比例

## 故障排除

### 常见问题

#### 问题1: 找不到数据文件

**错误信息**: `FileNotFoundError: 物流车配送距离.xlsx`

**解决方案**:

1. 确认数据文件存在于 `data/` 目录
2. 检查文件路径是否正确
3. 确认文件权限设置

#### 问题2: 内存不足

**错误信息**: `MemoryError`

**解决方案**:

1. 减少种群大小 (`population_size`)
2. 减少迭代代数 (`generations`)
3. 增加系统内存

#### 问题3: 优化无收敛

**现象**: 适应度值不下降或变化很小

**解决方案**:

1. 增加种群大小
2. 增加迭代代数
3. 调整变异率和交叉率
4. 检查数据质量

#### 问题4: 可视化失败

**错误信息**: matplotlib相关错误

**解决方案**:

1. 确认matplotlib版本正确
2. 检查中文字体设置
3. 确认输出目录权限

### 日志分析

系统会生成详细的运行日志，位置在 `output/system.log`。通过查看日志可以：

1. **跟踪执行过程**: 了解每个步骤的执行情况
2. **识别性能瓶颈**: 找出耗时最长的操作
3. **诊断错误原因**: 获取详细的错误信息
